# 初三内容规划（混合型）

## 规划目标

`grade9` 采用“前台按中考专题组织，底层保留教材映射”的混合结构。

这样做的目的有三点：

- 用户看到的是更符合初三复习节奏的专题学习入口。
- 内容层仍然能回答“这个知识点对应教材哪一章”。
- 后续既能按专题扩充，也能按教材来源、难度、题型和标签做筛选。

## 整体结构

建议将初三内容拆成 8 个专题模块：

1. 二次函数
2. 一元二次方程
3. 圆
4. 相似与锐角三角函数
5. 反比例函数与函数综合
6. 统计与概率
7. 几何综合
8. 代数综合与中考压轴入门

前台以“专题卡片”展示，底层继续沿用当前 `curriculum` 的 bundle 结构：

- `chapters.ts` 承担“专题元数据”
- `knowledgePoints.ts` 承担知识点清单
- `notes.ts` 承担专题讲义与复习笔记
- `questions.ts` 承担训练题与分层题组

## 专题设计

### 1. 二次函数

章节定位：
九年级高频核心专题，承担函数图像、解析式、最值、与几何/方程综合等大量中考题型。

建议知识点：

- 二次函数的概念与图像特征
- 一般式、顶点式、交点式
- 开口方向与对称轴
- 顶点坐标与最值
- 图像平移
- 与坐标轴交点
- 图像与参数关系
- 二次函数解析式求法
- 二次函数与面积问题
- 二次函数与最值问题
- 二次函数与几何综合

教材来源建议：

- 九上函数相关章节
- 中考复习专题补充

### 2. 一元二次方程

章节定位：
初三代数主干专题，为函数、应用题和综合题提供基础模型。

建议知识点：

- 一元二次方程的概念
- 配方法
- 公式法
- 因式分解法
- 根的判别式
- 根与系数关系
- 方程实际应用
- 一元二次方程与二次函数联系

教材来源建议：

- 九上一元二次方程章节

### 3. 圆

章节定位：
初三几何主干专题，是证明题、计算题和综合题的高频考点。

建议知识点：

- 圆的基本概念
- 圆心角、弧、弦
- 垂径定理
- 圆周角定理
- 圆内接四边形
- 点与圆、直线与圆的位置关系
- 切线性质与判定
- 与圆有关的计算
- 圆与相似、直角三角形综合

教材来源建议：

- 九下圆相关章节

### 4. 相似与锐角三角函数

章节定位：
连接几何证明、比例线段与解直角三角形，是初三几何和应用题的重要桥梁。

建议知识点：

- 比例线段
- 相似三角形判定
- 相似三角形性质
- 位似图形
- 锐角三角函数定义
- 特殊角三角函数值
- 解直角三角形
- 实际应用中的仰角俯角问题

教材来源建议：

- 八下相似
- 九下锐角三角函数

### 5. 反比例函数与函数综合

章节定位：
承接初二函数基础，进入初三综合场景后作为中档题和压轴题的常见入口。

建议知识点：

- 反比例函数图像与性质
- 反比例函数解析式求法
- 图像上的点与面积问题
- 一次函数、反比例函数、二次函数对比
- 函数综合问题中的分类讨论
- 函数与几何图形结合

教材来源建议：

- 八下反比例函数
- 初三专题复习补充

### 6. 统计与概率

章节定位：
偏稳定得分专题，适合做中考复习中的基础分与中档分模块。

建议知识点：

- 平均数、中位数、众数
- 方差与波动大小
- 频数分布表
- 统计图表分析
- 概率的意义
- 列举法求概率
- 树状图与列表法
- 概率与统计综合

教材来源建议：

- 八下统计与概率
- 初三总复习专题

### 7. 几何综合

章节定位：
承接三角形、四边形、相似、圆等知识，是中考综合证明与计算题的集中训练区。

建议知识点：

- 证明思路整理
- 常见辅助线策略
- 三角形与四边形综合
- 相似与圆综合
- 几何存在性问题
- 几何最值问题

教材来源建议：

- 七下到九下几何主干内容综合映射

### 8. 代数综合与中考压轴入门

章节定位：
作为初三收口专题，重点帮助学生建立“压轴题并非只能靠技巧”的分层训练路径。

建议知识点：

- 方程与函数综合
- 分类讨论的常见入口
- 数形结合
- 参数问题
- 代数最值问题
- 压轴题拆解方法
- 中考综合题阅读顺序

教材来源建议：

- 九年级复习综合专题

## 数据结构建议

建议先保持和当前 `grade7/grade8` 一致的基础结构，不急着改全局类型。

### chapters.ts

继续复用 `ChapterData`，但语义改成“专题”：

- `id`: `ch1` 到 `ch8`
- `title`: 专题名，如“二次函数”
- `subtitle`: 更细的复习定位
- `description`: 专题说明
- `sections`: 该专题下的知识点分组
- `isKeyChapter`: 标识是否为核心专题

### knowledgePoints.ts

继续复用 `KnowledgePointData`，但内容组织建议更贴近复习场景：

- `chapterId`: 对应专题
- `title`: 知识点名
- `content`: 一句话知识点定位
- `videoScript`: 作为短视频讲稿或口播提纲
- `example`: 对应该知识点的典型例题
- `exercises`: 基础练习和方法训练
- `formulas`: 常用结论、公式或方法提示

建议增加扩展字段时优先考虑：

- `sourceChapters`: 对应教材来源
- `tags`: 如 `函数`、`几何`、`压轴`、`高频`
- `examLevel`: 如 `基础`、`提升`、`冲刺`

如果当前全局类型不方便修改，可以先把这些信息折叠进 `content` 或 `formulas`，后续再扩类型。

### notes.ts

初三的 `notes` 不建议只沿用教材式讲解，建议改成“专题复习讲义”。

每个专题笔记至少包含：

- 专题定位
- 核心知识网络
- 常见题型
- 解题抓手
- 易错提醒
- 真题感知

### questions.ts

初三题目建议按复习场景分层：

- 基础题
- 方法题
- 综合题
- 压轴入门题

如果沿用现有 `QuestionData`，可以通过标签、难度和章节映射先承载，不必一开始重构题库模型。

## 首批实施建议

建议不要一口气做完 8 个专题，先做 3 个样板专题，把结构跑通：

1. 二次函数
2. 一元二次方程
3. 圆

这 3 个专题覆盖了：

- 代数主干
- 函数主干
- 几何主干

而且能较快验证下面几件事是否可行：

- 专题型 `chapters.ts` 是否顺手
- 知识点密度是否适合 3 分钟内容
- 笔记更偏复习讲义后是否更贴合初三定位
- 题目分层是否足够清晰

## 建议的交付顺序

### 第一阶段：搭骨架

- 建立 `grade9/chapters.ts`
- 建立 `grade9/knowledgePoints.ts`
- 先接通 3 个专题的最小可用数据
- 更新 `registry.ts`，让 `grade9` 可被启用

### 第二阶段：补内容

- 完成 3 个样板专题的笔记与题目
- 完成专题卡片与详情页内容校验
- 根据使用体验调整知识点粒度

### 第三阶段：扩专题

- 补齐剩余 5 个专题
- 增加按难度、标签、教材来源筛选
- 增加“综合复习”和“冲刺训练”导向入口

## 风险与取舍

### 1. 专题粒度过粗

如果每个专题过大，内容会显得笼统，视频脚本和题目也会过于分散。

建议：
优先保证知识点层足够细，而不是专题数量越多越好。

### 2. 过早重构全局类型

如果一开始就为 `sourceChapters / tags / examLevel` 改动全局类型，可能会把工作重心从内容建设转到架构重构。

建议：
先复用现有模型，等首批 3 个专题稳定后再评估是否扩字段。

### 3. 复习导向不够强

如果初三内容只是把教材章节换个名字，产品层会显得和初一初二差异不明显。

建议：
初三笔记与题目一定要更偏“复习抓手、题型方法、易错整理”，而不是纯教材讲解。

## 一句话结论

`grade9` 最适合做成“前台专题化、底层保留教材映射”的混合结构。

建议先用“二次函数 + 一元二次方程 + 圆”做第一批样板专题，把专题型骨架、知识点粒度和复习导向全部跑通，再扩到完整初三体系。
